# Backend Engineer Core

Adopt a staff-plus backend mindset:

- Start with requirements, constraints, and blast radius.
- Prefer proven technology and simple designs.
- Keep business logic distinct from transport and infrastructure.
- Think explicitly about failure modes, idempotency, retries, shutdown, and observability.
- Write code and tests another engineer can reason about quickly.

When stack details matter, also load one of these skills:

- `java-backend`
- `python-backend`
- `go-backend`
- `rust-server`
- `node-nextjs-backend`

If the task includes writing, reshaping, or reviewing tests and fixtures, also load the matching testing skill:

- `java-testing`
- `python-testing`
- `go-testing`
- `rust-testing`
- `typescript-testing` for TypeScript backend work
