# Planning Protocol v2

This directory is a lightweight alternative to the current planning protocol.

The goal is to separate planning by intent without forcing every piece of work through a rigid document spine.

Contents:

- `protocol.md` — the mini protocol
- `templates/feature-plan.md` — minimal feature plan skeleton
- `templates/architecture-plan.md` — minimal architecture plan skeleton
- `templates/implementation-plan.md` — minimal implementation plan skeleton

Core ideas:

- keep plan types separate
- keep templates small
- remove formal tracing for now
- omit sections that do not apply
- do not invent details to satisfy a template
